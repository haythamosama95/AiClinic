# Backend Test Coverage Catalog

Documents SQL verification suites in `backend/tests/`. Each suite runs via `psql` with `ON_ERROR_STOP=1`; failures raise when any result row has `passed = false`.

## How to read this document

- Tests are grouped by product domain (auth, org/branch, patients, appointments, safety).
- **Pass**: named result row is stored with `passed = true` and the script completes.
- **Fail**: result row fails or the script aborts.
- Scenarios listed here reflect **implemented** checks only; deferred items are noted separately.
